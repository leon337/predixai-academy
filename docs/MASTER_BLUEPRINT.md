# PredixAI Academy — Master Blueprint

## 1. Objetivo

Consolidar em um único documento a referência oficial de produto, aprendizagem, design e engenharia da PredixAI Academy.

## 2. Missão

Transformar conhecimento técnico complexo em aprendizagem clara, visual, prática e aplicável, medindo competências e não apenas consumo de aulas.

## 3. Produto

A PredixAI Academy será uma plataforma educacional web, responsiva e escalável para trilhas, cursos, módulos, aulas, quizzes, biblioteca, progresso e projetos práticos.

## 4. Hierarquia de conteúdo

```text
Trilha
└── Curso
    └── Módulo
        └── Aula
            └── Unidade de aprendizagem
```

## 5. Curso-piloto

**Integração com WhatsApp Business Platform**

Primeira entrega editorial:

- Módulo 1 — Fundamentos e Ecossistema;
- página web;
- livro PDF ilustrado;
- glossário;
- quiz;
- checklist;
- apresentação;
- materiais complementares.

## 6. Empresa fictícia

A empresa fictícia provisória é **VivaBem Serviços Integrados**. Ela será o caso progressivo usado para demonstrar atendimento, orçamento, agenda, operações, financeiro, CRM, WhatsApp Business Platform e agentes de IA.

## 7. Metodologia

Ciclo obrigatório:

```text
Contexto
→ Problema
→ Conceito
→ Analogia
→ Representação visual
→ Exemplo
→ Aplicação guiada
→ Prática independente
→ Feedback
→ Revisão
→ Entrega
```

## 10. Arquitetura técnica

### Frontend

- Next.js;
- React;
- TypeScript;
- App Router;
- componentes reutilizáveis;
- conteúdo em Markdown/MDX;
- design tokens.

### Backend e dados

- Supabase para autenticação, PostgreSQL, progresso e armazenamento;
- Row Level Security;
- dados de conteúdo versionados no GitHub;
- dados de usuário e progresso no banco.

### Deploy

- GitHub como fonte técnica;
- Linear como fonte operacional;
- Vercel para preview e produção;
- Supabase para dados e autenticação.

## 13. Critérios de qualidade

Toda entrega deve:

- possuir objetivo claro;
- ter critérios de aceitação;
- ser responsiva;
- ser acessível;
- usar componentes reutilizáveis;
- manter consistência visual;
- evitar conteúdo duplicado;
- ter evidência de teste;
- ser publicada apenas após validação.
